課題

ベクトル検索は類似テキストを見つけるが、
意図は理解しない

コーディングエージェントはコードを検索するが、コードを理解していない。

開発者のクエリ：
"決済処理に関連するファイルを探して"
ベクトル検索はコンテンツやエンベディングに「決済」を含むファイルを返すが、決済フローに不可欠なサービス層、バリデータ、エンティティ、テーブルクラスを完全に見逃す。
  • 関係性の認識なし — import、継承、メソッド呼び出しが見えない
  • 影響分析なし — 1ファイルを変更して、知らないうちに5つの依存ファイルを壊す
  • 構造的コンテキストなし — エージェントが十分に理解していないコードを変更する
  • 結果：デプロイの失敗、バグの混入、開発者の信頼低下

コードインテリジェンスを実現するグラフクエリの実例

このファイルに依存しているものは？
GRAPH code_graph_a
MATCH (dep:Files)-[:FileDependsOn]->(f:Files)
WHERE f.file_name = 'PaymentsService.php'
RETURN dep.file_name
AppControllerを継承しているクラスは？
GRAPH code_graph_a
MATCH (c:Classes)-[:ClassInherits]->{1,5}(p:Classes)
WHERE p.name = 'AppController'
RETURN c.name, c.kind
影響分析：UsersController変更時
GRAPH code_graph_a
MATCH (a:Files)-[:FileDependsOn]->{1,3}(f:Files)
WHERE f.file_name = 'UsersController.php'
RETURN a.file_name -- 影響を受けるノード
循環依存の検出
GRAPH code_graph_a
MATCH (a:Files)-[:FileDependsOn]->{2,10}(a)
RETURN DISTINCT a.file_name -- 循環参照
未使用クラスの検出
GRAPH code_graph_a
MATCH (c:Classes)
WHERE c.kind = 'class'
AND NOT EXISTS { MATCH ()-[]->(c) }
RETURN c.name -- 孤立クラス
メソッド呼び出しチェーン分析
GRAPH code_graph_a
MATCH (m1:Methods)-[:MethodCalls]->(m2:Methods)-[:MethodCalls]->(m3:Methods)
WHERE m1.name = 'processPayment'
RETURN m1.name, m2.name, m3.name
